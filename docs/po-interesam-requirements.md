## Рекомендуемый Вариант Архитектуры

### Вариант A

Главная страница + блок `Выберите свой маршрут по фестивалю` + `7` leaf-страниц. Отдельную hub-страницу можно добавить позже.

Плюсы:
- не создаёт лишний промежуточный экран в v1;
- даёт sharable и индексируемые тематические страницы;
- не превращает главную страницу в фильтр-систему вместо фестивального лендинга.

Минусы:
- меньше SEO-поверхности, чем при наличии отдельной hub-страницы;
- часть пользователей будет входить в тематические страницы только через главную.

Статус: `Не подтверждено пользователем`

### Вариант B

Главная страница + отдельная hub-страница `/po-interesam/` + `7` leaf-страниц.

Плюсы:
- максимальная SEO/GEO-поверхность;
- отдельная indexable-точка входа в семейство маршрутов;
- удобнее для внешних ссылок и llms.txt.

Минусы:
- требует ещё один уникальный редакционный слой, иначе hub станет thin-page;
- увеличивает объём вёрстки и контента в первой итерации.

Статус: `Не подтверждено пользователем`

### Вариант C

Только блок на главной странице без отдельных leaf-страниц.

Плюсы:
- минимальная стоимость внедрения.

Минусы:
- слабый SEO/GEO-эффект;
- нельзя удобно делиться конкретным маршрутом;
- превращает маршруты в декоративный блок, а не в новый сценарий входа.

Статус: `Не рекомендуется`

## Структура Тематической Страницы

- На каждой тематической странице есть свой hero, вводный блок, список событий, повторная навигация по маршрутам и footer. — Статус: `Не выполнено`
- Страница не должна быть копией главной программы с другим H1. — Статус: `Не выполнено`
- У страницы должен быть собственный вводный текст, основанный на source-описании маршрута, а не только набор карточек. — Статус: `Не выполнено`
- Каждое событие внутри одной тематической страницы показывается только один раз, даже если логически относится к нескольким внутренним секциям. — Статус: `Не выполнено`

### Hero Тематической Страницы

- В hero показывается один крупный миф за раз. — Статус: `Не выполнено`
- Источник текста для hero — только строки `Итоговая` из [mify_festivalya_po_kategoriyam.md](/workspaces/kdg80/Исходные%20данные/mify_festivalya_po_kategoriyam.md). — Статус: `Зафиксировано`
- Hero-маркер должен явно показывать, что перед пользователем миф или заблуждение, а не цитата спикера. — Статус: `Не выполнено`
- Под мифом показывается связанная лекция и спикер, который этот миф разбирает. — Статус: `Не выполнено`
- Нажатие на миф переводит на соответствующее событие. — Статус: `Не выполнено`
- Переходы между мифами должны быть дорогими и динамичными, но без layout shift: допустимы только `opacity` и `transform`, а не reflow-анимации. — Статус: `Не выполнено`
- Анимация должна быть прерываемой и уважать `prefers-reduced-motion`. — Статус: `Не выполнено`
- Для страницы `Паблик-токи и дискуссии` автоматическая myth-ротация не может считаться готовой, пока не будет собран ручной publish-ready пул мифов или согласован fallback-hero. — Статус: `В доработке`

### Список Событий На Странице

- Сначала идут редкие форматы, затем массив лекций. — Статус: `Не выполнено`
- Внутри каждого форматного раздела события отсортированы хронологически. — Статус: `Не выполнено`
- Рекомендуемый приоритет редких форматов: `Открытие фестиваля`, `Иммерсивный спектакль`, `Выставки`, `Экскурсия`, `Паблик-токи`, `Лекции`. — Статус: `Не подтверждено пользователем`
- Разделы по типам событий оформляются как крупные самостоятельные заголовки, а не как мелкие бейджи внутри бесконечной ленты. — Статус: `Не выполнено`

### Навигация По Темам

- В шапке тематических страниц нужен второй уровень навигации со всеми маршрутами. — Статус: `Не выполнено`
- Текущая тема должна быть явно отмечена как active. — Статус: `Не выполнено`
- На mobile эта навигация не должна ломать высоту первого экрана и должна быть тач-дружественной. — Статус: `Не выполнено`
- Логотип на тематической странице ведёт на главную страницу, а не в начало текущей страницы. — Статус: `Не выполнено`

### Конец Страницы

- Footer обязателен и считается частью сценария страницы, а не техническим хвостом. — Статус: `Не выполнено`
- Над footer рекомендуется добавить компактный блок `Сменить маршрут` с быстрыми переходами на другие темы. — Статус: `Не подтверждено пользователем`
- Одного footer как единственной повторной навигации по темам может оказаться недостаточно на длинной странице. — Статус: `Зафиксировано`

## Главная Страница

### Блок «Выберите свой маршрут по фестивалю»

- На главной странице появляется компактный блок ссылок на все тематические страницы. — Статус: `Не выполнено`
- Блок не должен выглядеть как второй hero или как ещё одна полновесная секция программы. — Статус: `Не выполнено`
- Основной текст блока: `Выберите свой маршрут по фестивалю`. — Статус: `Зафиксировано`
- Визуальный уровень блока должен быть ниже hero и ниже основной программы, но выше footer. — Статус: `Не подтверждено пользователем`
- Sticky-режим этого блока при скролле допустим только как улучшение, если он уступает место sticky-навигации по месяцам до начала программы. — Статус: `Не подтверждено пользователем`

### Главный Hero

- На главной странице myth-варианты входят в общую ротацию с сильными цитатами спикеров. — Статус: `Не выполнено`
- При показе myth-варианта фото спикера не показывается. — Статус: `Не выполнено`
- Миф должен быть явно помечен как миф или заблуждение. — Статус: `Не выполнено`
- Под мифом показывается строка о том, кто и на какой лекции будет разбирать это заблуждение. — Статус: `Не выполнено`
- Для myth-варианта ссылка на лекцию обязательна. — Статус: `Не выполнено`
- Автоматическая ротация на главной должна стартовать сама, без ручного запуска пользователем. — Статус: `Не выполнено`

## Переход В Программу И Поведение Лого

- На главной странице нажатие на лого ведёт к началу страницы. — Статус: `Не выполнено`
- На тематических страницах нажатие на лого ведёт на главную страницу. — Статус: `Не выполнено`
- CTA `Программа` на тематической странице ведёт на главную страницу к началу актуального по времени участка программы. — Статус: `Не выполнено`
- Если текущая дата находится до начала фестиваля, переход ведёт к первому месяцу программы. — Статус: `Не подтверждено пользователем`
- Если текущая дата находится внутри фестиваля, переход ведёт к ближайшему ещё не завершившемуся месяцу. — Статус: `Не подтверждено пользователем`
- После завершения фестиваля нужна отдельная договорённость: вести либо в архив, либо к последнему месяцу программы. — Статус: `В доработке`

## SEO И GEO

- Каждая тематическая страница должна быть indexable и иметь self-canonical. — Статус: `Не выполнено`
- Каждая тематическая страница должна иметь собственные `title` и `description`, а не шаблон от главной страницы. — Статус: `Не выполнено`
- Для тематических страниц нужен JSON-LD уровня `CollectionPage` или `WebPage` + `ItemList` + `BreadcrumbList`. — Статус: `Не выполнено`
- Тематические страницы должны попасть в `sitemap.xml` после запуска. — Статус: `Не выполнено`
- Тематические страницы должны быть добавлены в `llms.txt` после запуска. — Статус: `Не выполнено`
- Каждая тематическая страница должна содержать минимум один self-contained объясняющий блок, который отвечает, что это за маршрут и зачем он нужен пользователю. — Статус: `Не выполнено`
- Intro-тексты для страниц нельзя повторять между собой с косметическими заменами. — Статус: `Не выполнено`
- Если hub-страница `/po-interesam/` запускается без полноценного уникального контента, её лучше временно не индексировать. — Статус: `Не подтверждено пользователем`

## Риски И Подводные Камни

- Группы пересекаются, поэтому пользователь легко может попасть на duplicate-looking страницы, если они будут отличаться только набором карточек. — Статус: `Зафиксировано`
- Сейчас source-группировка даёт `56` уникальных строковых сущностей, тогда как publish-ready master-модель описывает меньший набор основных программных сущностей; это означает необходимость явного entity-mapping перед разработкой. — Статус: `Зафиксировано`
